# Экспорт плана: PDF / PNG / JSON (LKWkalk-qrd.15)

> Дизайн-раздел по директиве автономии: дефолты приняты письменно, без брейншторм-диалога.
> Аудитория: следующий агент + владелец. Дата: 2026-07-17.

## 1. Задача

`LKWkalk-qrd.15`: с экрана **Ladeplan** пользователь выгружает результат расчёта тремя способами.
Критерий приёмки бида: три экспорта дают файлы; JSON соответствует контракту движка;
PDF/PNG содержат **обе проекции** (вид сверху + вид сбоку) и **сводку**.

## 2. Принятые решения (дефолты)

| Решение | Выбор | Почему |
|---|---|---|
| **PDF** | Через **системный диалог печати** (`window.print()`), пункт «Als PDF speichern» | Печать A4 landscape уже вылизана (`theme.css`, батч `2ll`): 1 лист, обе проекции, сводка, штриховка прямыми `<line>` (Chrome не печатает `<pattern>`). Это уже и есть PDF-отчёт — вектор, шрифты, поля. Библиотека (jsPDF+svg2pdf ≈ 400 КБ в бандл) дала бы худший результат и второй набор багов вёрстки. |
| **PNG** | Композитный SVG (заголовок + мета + обе проекции + сводка + легенда) → `<img>` → `<canvas>` → `toBlob` | Без новых зависимостей. Растр по требованию: 2× плотность (ширина 2400 px). |
| **JSON** | `{ exportedAt, engineContractVersion, load, layout }`, скачивание файлом | `load` нужен для воспроизведения расчёта, `layout` — сырой результат по контракту. Версия контракта — чтобы потребитель (ERP/MCP) знал схему. |
| **Что экспортируется** | Текущий **отредактированный** `layout` (после ручных правок drag/rotate) | Экспорт = «то, что вижу на экране». |
| **UI** | В панели действий Ladeplan: `PDF` · `PNG` · `JSON` (secondary) + существующая `Drucken` (primary) | Без новых примитивов (выпадающее меню = свой click-outside и его известные грабли). `PDF` рядом с `Drucken` намеренно: одна кнопка «на бумагу», другая «в файл», обе через один диалог — у `PDF` есть `InfoHint` с пояснением. |
| **Имя файла** | `ladungsplaner-<slug(vehicle.name)>-<YYYY-MM-DD>.<ext>` | Узнаваемо в папке «Загрузки», сортируется по дате. |

**Новые зависимости: нет.**

## 3. Модуль `apps/web/src/lib/exportPlan.ts`

Чистые функции (тестируемые в jsdom) + тонкий браузерный слой:

- `slugify(name): string` — латиница/цифры/дефис, нижний регистр.
- `planFilename(vehicleName, kind, date): string` — `kind: 'json' | 'png'`.
- `buildPlanJson(load, layout, exportedAt): PlanExport` — объект экспорта.
- `inlinePaint(el)` — рекурсивно проставляет вычисленные `fill`/`stroke` вместо `var(--…)`
  (в сериализованном SVG вне документа CSS-переменные не резолвятся — картинка вышла бы чёрной).
- `buildCompositeSvg({ svgs, title, meta, figures, legend, width })` — разметка одного SVG-листа.
- `rasterize(svgMarkup, widthPx): Promise<Blob>` — `<img>` + `<canvas>`; **только браузер**
  (в jsdom нет canvas → покрывается ручной верификацией, не тестом).
- `downloadBlob(blob, filename)` — `URL.createObjectURL` → `<a download>` → `revokeObjectURL`.

## 4. Тесты (TDD, сначала падающие)

1. `slugify` / `planFilename` — пробелы и умляуты (`LKW Standard`, `Mega-Auflieger`), дата в имени.
2. `buildPlanJson` — содержит `load`, `layout`, версию контракта (`ENGINE_CONTRACT_VERSION`),
   переданный `exportedAt`; `JSON.parse(JSON.stringify(x))` круглым ходом равен исходному (сериализуем).
3. `buildCompositeSvg` — содержит оба вложенных `<svg>` в порядке «сверху, сбоку», заголовок,
   строки сводки и легенды; ширина/высота листа положительные.
4. `inlinePaint` — заменяет `fill="var(--s1)"` на вычисленное значение (computed style замокан).
5. `downloadBlob` — создаёт и отзывает objectURL, кликает по `<a>` с нужным `download`.
6. `LadeplanScreen` — три кнопки на экране; `JSON` вызывает скачивание (мок `URL.createObjectURL`);
   `PDF` вызывает `window.print()`.

## 5. Инварианты и границы

- Экспорт **ничего не считает**: только сериализует уже готовые `load`/`layout` (вся доменная логика — в ядре).
- Контракт движка **не меняется** (0.11.0) — это UI-фича, ADR не требуется.
- i18n: новые ключи `action.exportPdf`, `action.exportPng`, `action.exportPdfHint`
  (`action.exportJson` уже есть в словарях, до сих пор не использовался) — de + ru.
- Дизайн-токены: в композитном SVG цвета берутся из **вычисленных** значений темы, ни одного hex в коде.
- PNG-растеризация не покрывается автотестом (jsdom без canvas) → обязательна ручная проверка на проде.

## 6. Вне объёма

- Серверный рендер PDF, e-mail-отправка, экспорт в CSV/Excel, пакетный экспорт нескольких планов.
- 3D-вид в экспорте (3D вне MVP).

# Вид сбоку: глубина ряда по перекрытию проекций

> Дизайн-раздел по директиве автономии. Источник — наблюдение владельца (2026-07-17):
> *«вид сбоку иногда путает передний и задний ряд»*. Диагноз воспроизведён тестом (§2).
> Едет отдельно и первым: маленький, проверяемый, ни от чего не зависит.
> Парная спека — [склад 1:1 и магнит постановки](2026-07-17-warehouse-buffer-design.md).

## 1. Что не так сегодня

`sideRects` ([cutaway.ts:94-100](../../../apps/web/src/screens/components/cutaway.ts#L94-L100))
ранжирует стопки по глубине **только среди тех, у кого совпадает `x`**:

```ts
const ysByX = new Map<number, number[]>();
for (const s of stacks) { ...arr.push(s.y); ysByX.set(s.x, arr); }
for (const arr of ysByX.values()) arr.sort((a, b) => b - a);
// depth = ysByX.get(s.x)!.indexOf(s.y)
```

Но на виде сбоку перекрываются стопки с **пересекающимися интервалами** по длине, а не только с
равным `x`. Две стопки в разных рядах, стоящие «внахлёст» по длине, получают **обе** `depth = 0`:

- ни одна не притеняется — дальний ряд читается как передний;
- при равной глубине сортировка `sortedRects` ([CrossSection.tsx:111-112](../../../apps/web/src/screens/components/CrossSection.tsx#L111-L112))
  их не различает, порядок отрисовки остаётся исходным порядком `placements` — **дальняя стопка
  может закрасить ближнюю**.

Поэтому баг и «иногда»: на однородном грузе ряды выровнены по `x`, `ysByX` работает, всё верно.
Ломается на смешанных типах — там, где длины разные и ряды разъезжаются.

**Вторая, тихая беда того же кода.** Ранг считается внутри своего `x`, поэтому одиночная стопка,
которую никто не загораживает, всё равно получает `depth > 0`, если в её колонке `x` есть стопка с
бо́льшим `y`. Она притеняется без причины.

## 2. Воспроизведение

Кузов 13600×2430×2650, две стопки в разных рядах с перекрытием по длине и разными `x`:

| Стопка | `x` | длина | `y` | Ожидание |
|---|---|---|---|---|
| A | 0 | 1200 (0…1200) | 0 | дальняя → `depth = 1`, притенена |
| B | 600 | 1200 (600…1800) | 1600 | ближняя → `depth = 0` |

Факт: `{ rearDepth: 0, nearDepth: 0 }` — обе «передние».

## 3. Решение

### 3.1 Глубина = число стопок, реально загораживающих эту

Для стопки `s` глубина — количество стопок, которые **перекрывают её по длине** и стоят **ближе к
зрителю** (бо́льший `y`, по конвенции вида сбоку из [ADR 012](../../adr/012-loading-modes.md) и §9
HANDOVER-2026-07-16):

```ts
const overlaps1d = (a0, a1, b0, b1) => a0 < b1 && b0 < a1;
depth(s) = stacks.filter((o) => o !== s && o.y > s.y &&
                                overlaps1d(s.x, s.x + s.w, o.x, o.x + o.w)).length;
```

`depth = 0` теперь означает ровно «эту стопку ничто не загораживает» — и одиночная стопка в дальнем
ряду перестаёт притеняться без причины.

Стоимость — O(n²) по числу напольных стопок (45 на демо, сотни в пределе). Ничтожно на фоне отрисовки;
экономить структурой (интервальное дерево) — преждевременно.

### 3.2 Порядок отрисовки — по `rowY`, а не по глубине

`CrossSection` сортирует по **возрастанию `y`** (дальние раньше ближних), тай-брейк по `x` — ради
детерминизма (принцип проекта). Это корректно **глобально**: из любых двух перекрывающихся стопок
та, что ближе (бо́льший `y`), рисуется позже и закрывает дальнюю.

Сортировка по `depth` для этого не годится и заменяется: «загорожена двумя» не означает «дальше, чем
загороженная одной» — это не порядок.

### 3.3 `D2` — дальний ряд гасится заливкой, не контуром

Сегодня вся группа дальней стопки гасится `opacity 0.4`, включая контур и штриховку. На демо
четвертьпаллеты (6 ярусов = 864 мм из 2650) в дальнем ряду почти не видны. Вид сбоку по парной спеке
становится **первой** секцией плана, и его читаемость дорожает.

Решение: `opacity` уходит с группы; гасится **только заливка и штриховка** дальних стопок
(`fillOpacity 0.16 → 0.06`, штриховка `0.8 → 0.25`), контур остаётся в полную силу. Дальний ряд
читается как «за» — но читается.

## 4. Контракт и границы

Изменения **только в UI-проекции** (`cutaway.ts`, `CrossSection.tsx`). Движок, контракт и раскладка не
трогаются: `sideRects` — чистая функция над готовым `Layout`. Версия контракта не меняется.

## 5. Тесты (TDD, до кода)

`cutaway.test.ts`:
1. **Баг из §2**: разные `x`, перекрытие по длине → у дальней `depth = 1`, у ближней `depth = 0`.
2. **Ложное притенение**: одиночная стопка в дальнем ряду, никто не перекрывает → `depth = 0`.
3. **Регресс**: совпадающие `x` (сегодняшний рабочий случай) — ранги прежние.
4. **Детерминизм**: тот же вход → тот же порядок (тай-брейк по `x`).

`CrossSection.test.tsx`:
5. Порядок отрисовки — по возрастанию `rowY`: дальняя стопка идёт в DOM раньше ближней.
6. Дальняя стопка: контур в полную силу, заливка приглушена (D2).

Существующие 365 тестов — зелёные без правок ожиданий (кроме тех, что фиксировали старое притенение).

## 6. Что НЕ входит

- Перестановка секций, склад, магнит — парная спека.
- Подпись PNG из `data-cutaway` — там же (она становится нужна вместе с перестановкой).
- Плотность режима «Сбоку» — потолок shelf-модели, разобран в `u02`, лечится `i1p`.

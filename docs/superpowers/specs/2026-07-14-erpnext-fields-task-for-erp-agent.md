# Задание для ERPNext-агента: кастом-поля габаритов паллет

Статус: TASK — для агента, ведущего **настройку ERPNext и миграцию с minisys**. bd: см. `LKWkalk`
(создаётся отдельной задачей). Приоритет: можно **позже** (после переноса ERPNext в облако Schäfer).

> Полный дизайн и обоснование: [спека кастом-полей](2026-07-14-erpnext-dimension-fields-design.md).
> Это операционное задание «что сделать в ERPNext», спека — «почему так».

## Контекст

Приложение **Ladungsplaner** (расчёт загрузки грузовика паллетами) импортирует заказы из ERPNext и
берёт **габариты груза** из кастом-полей. Товары ERPNext (группа `Ladungsträger Artikel`) — это сами
паллеты. Сейчас габаритных полей нет (габариты только в названии, парсить их приложение НЕ будет).

## Что сделать в ERPNext

Создать **три кастом-поля** (тип **Int**, значение в **целых миллиметрах**):

| fieldname | label (de) | Тип |
|---|---|---|
| `custom_length_mm` | Länge (mm) | Int |
| `custom_width_mm`  | Breite (mm) | Int |
| `custom_height_mm` | Höhe (mm)  | Int |

1. **DocType `Item`** — добавить три поля (секция «Ladungsplaner / Abmessungen»). Это мастер-значения
   (габариты = свойство паллеты). Проставить их у актуальных товаров-паллет (при миграции с minisys —
   маппить из соответствующих полей minisys, если они там есть).
2. **DocType `Sales Order Item`** — добавить те же три поля, **Read Only**, каждому задать
   **Fetch From**:
   - `custom_length_mm` → `item_code.custom_length_mm`
   - `custom_width_mm`  → `item_code.custom_width_mm`
   - `custom_height_mm` → `item_code.custom_height_mm`
   Так значения копируются в строку заказа при выборе товара, и приложение читает их одним запросом.

## Важные нюансы

- **API-доступ:** приложению нужен API key/secret ERPNext с правом **чтения Sales Order + Sales Order
  Item** (сейчас у тестового токена был `403` на Sales Order Item — выдать доступ). Секреты передаются
  владельцу для настройки в Coolify (env), в git не кладутся.
- **`fetch_from` копирует при выборе товара, не задним числом:** старые заказы (созданные до полей)
  в строке будут пустыми — приложение пометит их как ручной ввод. При желании — разовый бэкфилл строк
  старых заказов скриптом (опционально).
- **Только габариты.** Правила груза (штабелирование/вложение/вращение) в ERPNext НЕ выносим — они
  задаются в приложении.
- Проверка: после настройки открыть любой Sales Order — в строках должны появиться заполненные
  `custom_*_mm`; приложение при импорте вернёт позиции с `dimensionsSource='erpnext-field'`.

## Definition of Done

- Три поля на `Item` и `Sales Order Item` (с `fetch_from`), заполнены у актуальных паллет-товаров.
- API-доступ к Sales Order/Item выдан, ключи переданы владельцу.
- На тестовом заказе приложение импортирует позиции с габаритами (не `manual`).

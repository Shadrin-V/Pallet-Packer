# Пользовательские пресеты паллет (LKWkalk-4bj.4)

> Спека T4 эпика `LKWkalk-4bj` (Ladeplan v2), 2026-07-16. Аудитория: агент Claude Code + диспетчер
> claude.ai. Процесс: docs-first → TDD. Брейншторм-диалог отменён (директива автономии) — дефолты
> ниже приняты письменно; ломающих изменений контракта нет.

## Контекст

Встроенные пресеты паллет (`PALLET_PRESETS`: EPAL 1/2/3/6, Viertelpalette) покрывают стандарт, но
владелец возит и нестандартные габариты (собственные короба, обрезные пакеты). Сейчас такие размеры
приходится вбивать руками при каждом расчёте. Задел уже в `main`:
`apps/web/src/data/userPresets.ts` (`loadUserPallets` / `addUserPallet` / `removeUserPallet` /
`isUserPreset`, localStorage `ladungsplaner.palletPresets`, ключи `user-<uuid>`) + i18n-ключи
`setup.savePreset` / `setup.deletePreset`. Осталась проводка в UI.

## Решения (дефолты приняты)

- **Хранилище — localStorage браузера** (ADR 007), рядом с черновиком формы. Это справочные данные
  формы, **не источник истины**: импорт ERPNext перезаписывает форму, но каталог пресетов не трогает.
  «Zurücksetzen» чистит черновик (`ladungsplaner.setup`), но **не** удаляет сохранённые пресеты —
  иначе пользователь терял бы свой справочник при каждом сбросе формы.
- **Идентификация пресета в дропдауне — по габаритам** (ДxШxВ), как и для встроенных: список
  `[custom, ...PALLET_PRESETS, ...userPallets]`, поиск текущего значения — по объединённому списку.
  Это сохраняет уже существующую конвенцию (выбранный пресет = совпадение размеров).
- **Дедупликация — по габаритам**: сохранение размеров, уже сохранённых пользователем, заменяет
  запись (не плодит дубли). Совпадение со встроенным пресетом → кнопка сохранения не показывается.
- **Имя пресета** — из поля «Bezeichnung» позиции (`p.name`), фолбэк `L×W×H`. Отдельного диалога
  ввода имени нет: лишний модал ради поля, которое уже есть в строке.
- **Место кнопок — раскрывающаяся панель details** позиции, не строка: строка позиции и так на грани
  по ширине (подводный камень с длинными RU-лейблами).
  - «Als Preset speichern» (`setup.savePreset`) — видна, когда габариты заданы и не совпадают ни с
    одним пресетом (встроенным или пользовательским).
  - «Preset löschen» (`setup.deletePreset`) — видна, когда текущие габариты = пользовательский пресет
    (`isUserPreset`).
- **Область видимости каталога — весь экран «Настройка»**: `userPallets` живёт в `SetupScreen` и
  прокидывается через `OrderCard` в `PositionRow`, чтобы пресет, сохранённый в одной позиции, сразу
  появлялся в дропдаунах всех остальных.

## Границы

- Контракт движка не меняется: пресет — это только заполнение полей `CargoType` в форме.
- Синхронизации между устройствами/пользователями нет (localStorage). Серверный каталог — возможный
  будущий бид, вне T4.
- Импорт/экспорт каталога пресетов JSON — вне T4 (в MVP-спеке это относится к справочникам движка).

## Тесты (TDD)

1. Сохранение: задать нестандартные габариты → «Als Preset speichern» → пресет появляется в дропдауне
   `Ladungsart` (в т.ч. в другой позиции того же экрана).
2. Персист: пресет переживает ремаунт `SetupScreen` (localStorage).
3. Удаление: выбранный пользовательский пресет → «Preset löschen» → исчезает из дропдауна.
4. Кнопка сохранения не показывается для габаритов встроенного пресета (EPAL 2).

# Ручной поворот стопки в виде сверху (LKWkalk-4bj.5)

> Спека T5 эпика `LKWkalk-4bj` (Ladeplan v2), 2026-07-16. Процесс: docs-first → TDD. Дефолты приняты
> письменно (директива автономии); ломающих изменений контракта движка нет — правка чисто в UI.

## Контекст

Вид сверху уже допускает ручную правку раскладки: `moveStack` (`components/dragLayout.ts`) двигает
стопку по сетке 100 мм и **отклоняет** результат, нарушающий геометрию (возвращает исходный `Layout`).
Развернуть стопку на 90° пользователь пока не может — а именно этого не хватает, чтобы вручную
дожать раскладку (повернуть паллету в кармане, выровнять ряд под борт).

## Решения (дефолты приняты)

- **Поворот — только вокруг вертикальной оси** (`lwh` ↔ `wlh`), для правил `yawOnly` и `full`.
  Правило `none` → аффорданс не показывается, `rotateStack` возвращает исходный layout.
  Обоснование: ADR 013 — упаковщик MVP использует только yaw-подмножество, опрокидывание на грань
  отложено. Опрокидывание меняет `dz`, а значит требует пересчёта z всех ярусов стопки (координаты
  ярусов посчитаны движком под исходную высоту) — это не ручная правка, а перерасчёт. Если
  опрокидывание понадобится — отдельный бид вместе с yaw-расширением упаковщика.
- **Якорь поворота — угол `(x, y)`**, футпринт меняет `dx`↔`dy`. Центр-сохраняющий поворот дал бы
  дробные/несглаженные координаты (внутри — целые мм, сетка 100 мм) и чаще упирался бы в борт.
- **Отклонение как у `moveStack`:** перекрытие футпринта с любой другой стопкой ИЛИ
  `findGeometryViolations ≠ []` → возвращаем исходный `Layout` (та же ссылка), правка не применяется.
  Явная проверка футпринтов нужна, потому что валидатор пропускает пары одной колонки
  (`cargoTypeId+x+y`).
- **Стопка со смешанной ориентацией ярусов или с опрокинутой ориентацией** (не `lwh`/`wlh`) —
  поворот отклоняется. Такие стопки MVP-упаковщик не строит; это защита, а не сценарий.
- **Аффорданс — по выделению:** клик по прямоугольнику в виде сверху (нажатие без перетаскивания,
  порог 40 мм) выделяет стопку → у её верхнего-правого угла появляется кнопка поворота
  (`role="button"`, `aria-label` = i18n `ladeplan.rotateStack`). Повторный клик по стопке или клик по
  фону снимает выделение. Выделение и кнопка — `print:hidden` (в бумажный Ladeplan не попадают).
- **Модуль:** `components/dragLayout.ts` → **`components/editLayout.ts`** (`moveStack` + `rotateStack`
  + общие хелперы): файл теперь про ручное редактирование раскладки в целом, а не только про drag.

## Границы

- Метрики не меняются: поворот не меняет число размещённых единиц (`totalPlaced`, `usedFloorPositions`).
- Поворот отдельного яруса (не всей стопки) — вне T5: ярус нельзя развернуть, не сломав колонку.
- Автоматический подбор ориентации — это T6 (`4bj.6`, умная раскладка), не сюда.

## Тесты (TDD)

1. `rotateStack` принимает поворот в свободное место: ориентация всех ярусов `lwh`→`wlh`,
   `findGeometryViolations = []`, число placements не меняется.
2. Отклоняет поворот, при котором повёрнутый футпринт налезает на соседнюю стопку (та же ссылка).
3. Отклоняет поворот, выводящий груз за габариты кузова.
4. Отклоняет поворот для позиции с `rotation: 'none'`.
5. `CrossSection`: клик по стопке в виде сверху показывает кнопку поворота; клик по ней зовёт
   `onRotateStack` с выделением; в виде сбоку кнопки нет.

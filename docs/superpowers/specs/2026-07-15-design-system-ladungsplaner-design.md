## 1. Цель и контекст

Проработать **полную дизайн-систему** приложения Ladungsplaner (не косметика): токены, типографика,
сетка/спейсинг, компоненты и их состояния, визуализация разрезов, печать A4. Достаточную, чтобы по ней
собрать эталонные экраны на React (apps/web) без дизайнерских решений «на ходу».

Отправная точка: авторские референс-макеты Claude ([`setup-reference.html`](../../lovable/setup-reference.html),
[`ladeplan-reference.html`](../../lovable/ladeplan-reference.html)) — нравятся; реализация в Lovable —
отвергнута (ADR 015), за ориентир не берётся.

Владелец дополнительно дал референсы-сайты: **smaut.tech**, **apple.com/airpods-max** (исполнение/craft, не
палитра), **perplexity.ai**. Общий вектор — безрамочность, много воздуха, крупнее иерархия, приглушённый
акцент. Основной цвет — фирменный, с **holz-schaefer.de**.

## 3. Направление (сводка характера)

**D · Clean Tech.** Холодная почти-белая поверхность с лёгким шалфейным уклоном нейтралей (не «безликий
серый»); поверхности разделяются мягкой тенью и воздухом, а не рамками; сдержанное движение
(Apple-restraint); глубокий фирменный зелёный как единственный тёплый-тёмный акцент. Читается «дорого,
современно, software-grade».

**Двойная плотность** реализуется одними токенами (разный padding-пресет), без отдельных компонентов:
«Настройка» компактнее, «Ladeplan» просторнее.

## 4. Фундамент: цвет и токены

Полные значения — в [`docs/design/theme.css`](../../design/theme.css). Ключевое:

- **Бренд:** `--brand #104F25`, `--brand-strong #0C3D1D` (hover), `--mint #71BF87`, `--sage #C3D1C7`.
- **Нейтрали (холодные, шалфейный уклон):** `--paper #F5F8F6` · `--card #FFFFFF` · `--sub #EBF0EC` ·
  `--ink #14201B` · `--muted #55605B` · `--faint #93A099` · `--line #E5EBE7` · `--line-strong #D5DDD8`.
- **Семантика (отдельно от бренда):** `--success`, `--warning #B8860B`, `--danger #B3261E`, `--info #1565C0`.
  Правило: семантика ≠ бренд-акцент, и наоборот.
- **Данные заказов:** `--s1…--s8`, **каждый всегда в паре со штрихом** (ЧБ-печать + дальтонизм).
- **Скругления:** `--r-card 12` · `--r-ctl 8` · `--r-pill 18`; тени мягкие, холодные, с зелёным подмесом.

Принцип: **ни одного hex в JSX — только токены**; ребренд = правка одного `theme.css`.

## 5. Типографика (Inter)

Модульная фиксированная шкала (титул 21/650, H2 16/600, eyebrow 11/600/UPPER/+.13em, label 10/600/UPPER,
тело 13–14/400, значение 13–14/600 tabular, формула 12.5 mono, caption 11.5/500). Заголовкам
`text-wrap: balance`; ширина текста ~65 символов. Полная таблица — в каноническом файле.

## 6. Сетка, спейсинг, адаптив

- Спейсинг-шкала 4px-based: `4·8·12·16·20·24·32·48`; отступы через flex/grid + `gap`.
- Контейнер десктоп ~1120px, поля 20–24px, межсекционный воздух 24–32px.
- Брейкпоинты: **≥1024** одна строка позиции · **640–1023** сжатие/перенос контролов · **<640**
  позиция → вертикальная карточка, primary-действие липнет снизу.

## 7. Компоненты и состояния

У каждого компонента заданы `default / hover / focus-visible / active / disabled / error` — только на
токенах. Рецепты (кнопка, поле-с-единицей, select, segmented Ent/Ver, chip, card, order card, position
row, add-bar, служебные: toast/empty/skeleton/modal/tooltip) — в каноническом файле, раздел «Компоненты».

Доступность: видимый focus-ring везде (mint-ring); контраст ≥ 4.5:1; идентичность заказа = **цвет И штрих**.

## 8. Визуализация данных (разрезы)

Inline-SVG в мм-координатах из `Layout`/`orientedDims`; фон-калька + сетка 1000 мм; стапель = `rect`
с паттерном цвета заказа (tint .16 + штрих .65) + stroke + подпись `×N`; 8 паттернов-штрихов в `<defs>`;
виды сверху и сбоку; легенда обязательна (цвет+штрих+Auftrags-ID). Данные-цвета ≠ бренд-акцент.

## 9. Печать A4 (только Ladeplan)

`@media print`: A4-портрет, UI-хром скрыт; состав — шапка (заказы/машина/дата) → разрезы сверху+сбоку →
легенда → сводка заполнения; `print-color-adjust: exact`; `break-inside: avoid` для разрезов и легенды;
всегда светлая, ЧБ-читаемость через штрихи.

## 10. Иконки, движение, тон

- **Иконки:** Lucide (open-source, линейный, stroke ~1.5, `currentColor`); эмодзи-иконки не используем.
- **Движение:** сдержанно; переходы 120–180ms ease-out; всё под `prefers-reduced-motion: reduce`.
- **Тон (de/ru):** активный залог; кнопка называет действие; ошибки объясняют что не так и как исправить;
  ни одной строки в коде — только ключи i18n (`@shadrin-v/i18n`, de по умолчанию + ru).

## 11. Non-goals / отложено

- Тёмная тема (позже; токены путь оставляют).
- Многостраничный брендированный печатный отчёт для клиента (вариант B печати — вне этой итерации).
- Экспорт PDF/PNG — отдельная задача; дизайн-система его не блокирует.

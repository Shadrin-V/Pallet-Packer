## 2. Что сделано

### `1t2` — вид сбоку путал передний и задний ряд (закрыт)

Спека [`specs/2026-07-17-side-view-depth-design.md`](specs/2026-07-17-side-view-depth-design.md).

Глубина ранжировалась только среди стопок с **совпадающим `x`**, но проекция перекрывается по
**интервалу** длины. Стопки в разных рядах внахлёст получали обе `depth = 0` → дальняя не притенялась
и могла закрасить ближнюю. «Иногда» — потому что на однородном грузе ряды выровнены по `x`.

Глубина теперь = число стопок, реально загораживающих эту; порядок отрисовки — по возрастанию `rowY`
(сортировка по глубине **некорректна**: это счётчик, а не порядок). D2: дальний ряд гасится заливкой,
контур в полную силу — при `opacity 0.4` низкие дальние стопки исчезали (область ~6 200…7 800 мм на
проде была пустой белой).

### Эпик `a5b` — склад 1:1 и магнит (закрыт)

Спека [`specs/2026-07-17-warehouse-buffer-design.md`](specs/2026-07-17-warehouse-buffer-design.md),
план [`plans/2026-07-17-warehouse-buffer.md`](plans/2026-07-17-warehouse-buffer.md),
[ADR 020](../adr/020-magnet-drop-resolution.md).

- Порядок секций: **сбоку → сверху → склад**. Подпись PNG — из `data-cutaway` (была из массива по
  индексу DOM: перестановка молча перепутала бы подписи в файле, уходящем другому человеку).
- **Склад** (`WarehouseFloor`) вместо полосы карточек: квадраты **1:1** с кузовом, общий примитив
  `StackShape` с видом сверху. Кнопка `⟳` ушла → поворот = жест кузова (выделить → `RotateHandle`),
  плитки фокусируемы (`tabIndex`/Enter). Погрузчик сверху, 1:1, только где не стоит на грузе.
- **Магнит** `resolveDrop` в ядре (контракт 0.13.0): ближайшее допустимое место с **прижимом
  вплотную**; живой призрак зелёный/красный + красные контуры мешающих стопок **до отпускания**.
- **D1+D3:** `Metrics` снят с листа, объём — пятой фигурой мета-полосы. Полоса — единственный
  источник цифр.

## 3. Решения владельца этой сессии (не пересматривать без него)

| Вопрос | Решение |
|---|---|
| Порядок секций | сбоку → сверху → склад |
| Раскладка склада | ряды с переносом, 1:1, ширина = кузову (в глубину растёт свободно) |
| Толерантность | магнит **+ прижим вплотную** (щель 60 мм закрывается) |
| Обратная связь | живая подсветка: зелёный/красный + кто мешает |
| Фон склада | погрузчик **сверху**, 1:1, без штабелей на полу |
| Пункты анализа UI | D1 + D2 + D3; `rgv.8` — **не берём** |

## 5. Подводные камни, добытые этой сессией

**Масштаб 1:1 — равенства `viewBox` НЕДОСТАТОЧНО.** Поймал только Chrome: свои `px-4` и рамка у секции
склада сделали svg на 34 px у́же колонки — `viewBox` совпадал, масштаб нет (0.09632 против 0.09882
px/мм). Правила: **между секцией и колонкой ни горизонтальных отступов, ни рамки**; **прокрутки у
склада нет** (полоса прокрутки на Windows/Linux сужает svg и уводит масштаб). Юнит-тест сторожит
только `viewBox`; пиксели — исключительно в браузере.

**Глубина — счётчик, а не порядок.** Сортировка по `depth` рисует дальнюю стопку поверх ближней, если
дальнюю загораживает одна соседка, а ближнюю — три. Порядок задаёт **`rowY`**. На двух стопках баг
не воспроизводится — нужна фикстура из пяти (см. `CrossSection.test.tsx`).

**Двойной снап ломает прижим.** `resolveDrop` возвращает позицию впритык на любом миллиметре
(Sonderpalette 1340 мм не даёт круглых кромок). Снап применяется **только к прицелу**, до магнита;
результат идёт в строгий `moveStack`/`placeStack` как есть. Поэтому `editLayout.ts` сведён к сетке.

**TDZ при чтении рефа во время рендера.** `tilePreview` считается в теле компонента, а `sheetRef`
объявлялся ниже → `Cannot access 'sheetRef' before initialization`. Поймали тесты, не typecheck.
Рефы, читаемые во время рендера, объявлять **выше** читателей.

**Puppeteer: `mouse.move` работает в координатах ВЬЮПОРТА.** План лежит далеко ниже сгиба — без
`scrollIntoView` указатель до него не доезжает, и жест молча не происходит (я потратил на это заход,
решив, что сломан продукт). Цель для drag брать **из DOM**, а не «на глаз».

**Плитки склада — стопки, а не паллеты.** Считать в проверках **единицы** (`warehouse-count`), а не
число плиток: стопки одного типа сливаются в одни плитки, и число плиток не меняется.

**Демо-вариант 3 — перегруз, кузов забит.** Ставить туда нечего: чтобы проверить зелёный призрак,
надо сначала **вынуть** стопку (появится дыра ровно нужного размера).

**Остаются в силе** (из вечернего снимка §5 и [2026-07-16](HANDOVER-2026-07-16.md) §9): PNG-экспорт
берёт разрезы только по `data-cutaway`; `resolveCssVars` обязателен; `downloadBlob` отзывает
objectURL через `setTimeout 0`; ориентация плитки — по типу груза; drag HTML→SVG на глобальных
`window`-слушателях + `getScreenCTM`; React — не setState внутри updater'а; Chrome не печатает SVG
`<pattern>`; click-outside на `click`; ручной поворот только yaw; `findGeometryViolations` пропускает
пары одной колонки; `dist` в .gitignore; **git checkout+merge+push по отдельности**; эталоны
**EUR 34 / GB 20 / none 33**.

## 2. Архитектура

Монорепо (npm workspaces `packages/*` + `apps/*`):
```
packages/engine    @shadrin-v/engine  — headless движок 2.5D (контракт 0.9.0). Работает В БРАУЗЕРЕ.
packages/i18n      @shadrin-v/i18n     — локали de/ru, форматирование. Строки — только ключи.
packages/contracts @shadrin-v/contracts— DTO API (граница web↔server).
apps/web           Vite React SPA      — два экрана (Настройка, Ladeplan), одна страница.
apps/server        Fastify + better-sqlite3 — персистенс + адаптер ERPNext + отдаёт SPA.
```
- **Движок в браузере** (ADR 001): SPA сама зовёт `calculateLayout`, `computeStack`, `orientedDims`,
  `findGeometryViolations`. Сервер тонкий.
- **Шов `DataProvider`** (`apps/web/src/data/`): единственный интерфейс данных. Сейчас `HttpDataProvider`
  → `apps/server`. B→A: замена реализации на Frappe без правки экранов (ADR 015).
- **Персистенс:** SQLite (`apps/server`, том `/app/data`), план = снимок `Load`+`Layout`+ref заказов.
- **i18n с первого коммита** (ADR 006), **токены дизайна** (design-system) — ни одного hex в JSX.
- **Геометро-инвариант:** `findGeometryViolations(load, layout) === []` на каждом результате.

## 3. Экраны (apps/web)

- **Настройка** (`screens/SetupScreen.tsx`): строка кузова + **пресеты фур** (Standard/Extra-hoch 2800/
  Mega 3000/Wechselbrücke/Frigo + «Eigene Maße»), карточки заказов (зоны), компактные строки-позиции —
  пресет-палета, размеры, кол-во (влево, без спиннера), segmented Ent/Ver, вращение, live-чип штабеля,
  раскрытие правил (режим вложения, Δh/h_д, maxNested, allowUnpairedTop, maxTiers, tooltip у «Stapelbar»),
  **живая формула** + **схема штабеля** (`StackDiagram`, в цвет заказа). Панель настроек — **аккордеон**
  (одна открыта на заказ) + **click-outside** закрывает, авто-раскрытие на «Verschachtelt». Дубль
  «+ Auftrag» внизу. **Persistence** (заказы+кузов в `localStorage`) + кнопка «Zurücksetzen». Строит `Load`.
  **Пользовательские пресеты паллет** (`4bj.4`): каталог `userPallets` (`data/userPresets.ts`,
  localStorage `ladungsplaner.palletPresets`) живёт в `SetupScreen` → `OrderCard` → `PositionRow`;
  дропдаун «Ladungsart» = `[custom, ...PALLET_PRESETS, ...userPallets]`, поиск текущего значения —
  по совпадению ДxШxВ в объединённом списке; кнопки «Als Preset speichern» (имя из поля позиции,
  фолбэк `L×W×H`) / «Preset löschen» — в панели details. «Zurücksetzen» чистит черновик формы, но
  **не** каталог пресетов.
- **Ladeplan** (`screens/LadeplanScreen.tsx`): бренд-шапка (логотип `public/logo.svg`) + мета-полоса
  (Fahrzeug/Aufträge) + фигуры Paletten/Stellplätze/Auslastung; два разреза SVG в мм
  (`components/CrossSection.tsx` + `cutaway.ts`) — вид сверху с ×N и **ручной правкой раскладки**
  (`editLayout.ts`: `moveStack` drag + `rotateStack` поворот, `4bj.5`),
  вид сбоку **силуэт-бар на штабель + задние ряды тусклее** (глубина по x). **Справка по составу**
  (`Legend.tsx` + `orderBreakdown.ts`) — цвет+штрих, артикул с габаритами L×W×H и кол-вом. **Печать A4

- **Одна страница** (`App.tsx`): SetupScreen всегда смонтирован (`print:hidden`), Ladeplan ниже при
  результате → состояние не сбрасывается; **план сохраняется в `localStorage` ('ladungsplaner.load')**,
  layout пересчитывается при загрузке. Переключатель языка **DE|RU** (`ui/LocaleSwitch.tsx`, localStorage).
- Дизайн-система: **Direction D**, бренд Holz Schäfer (`docs/design/design-system.md` + `theme.css` →

## 4. ERPNext (интеграция, вариант B)

- Адаптер `apps/server/src/erpnext/adapter.ts`: `importOrder` (Sales Order → зона), `searchOrders`,
  token-auth. Габариты — из **кастом-полей** `custom_length_mm/width/height` строки Sales Order
  (`fetch_from` Item); **парсера из названия нет** (по спеке). Провенанс `erpnext-field` / `manual`.
- REST `/api/orders`, `/api/orders/:id` с guard **`ERR_ERPNEXT_UNCONFIGURED` (503)**, когда секреты не
  заданы. **Сейчас ERPNext в локальном тест-режиме** — секреты не заданы, `/api/orders` отдаёт 503, это
  нормально. Поля в ERPNext создаст ERP-агент (задача `LKWkalk-k06`, спека
  `docs/superpowers/specs/2026-07-14-erpnext-fields-task-for-erp-agent.md`).

## 6. Как вести работу (рабочий процесс)

- **Docs-first:** изменилось требование/API — сначала spec/ADR/contract, потом код.
- **Процесс:** фича → `superpowers:brainstorming` → спека (`docs/superpowers/specs/`) → `writing-plans`
  (`docs/superpowers/plans/`) → TDD. Баг → `systematic-debugging`.
- **Автономия (память `autonomy-directive-2026-07-10`):** beads team-maintainer — авто commit/push/
  `bd dolt push`/close после зелёных гейтов; **ветка-на-задачу → merge в main сам**.
- **Гейты (перед merge):** `npm test` · `npm run lint` · `npm run typecheck` · `npm run build --workspace apps/web`
  · `docker build` (при изменении web/server). При правке `packages/engine` или `packages/i18n` — пересобрать
  их dist (`npm run build --workspace @shadrin-v/<pkg>`), т.к. web/server импортируют dist (dist в .gitignore).
- **Beads:** `bd ready` / `bd show <id>` / `bd update --status` / `bd close`. Эпик = группировка (НЕ вешать
  blocking-dep задача→эпик, иначе задачи блокируются).
- **Верификация live:** после merge — push `main` + `production`; Coolify авто-деплоит (webhook), ждать
  ~2–3 мин → проверить бандл на проде (скачать `/assets/index-*.js` в файл и `grep` маркеры).

## 9. Известные подводные камни

- **Coolify авто-деплоит** на push в `production` (webhook `la7` настроен) — Redeploy вручную не нужен.
- **dist в .gitignore** — на чистой машине сперва `npm ci` + сборка пакетов, иначе web/server-тесты не
  видят `@shadrin-v/engine`/`i18n`.
- **macOS case-insensitive FS** — не создавать в одной папке файлы, различающиеся только регистром
  (`CrossSection.tsx` vs `cutaway.ts` — ок; `crossSection.ts` vs `CrossSection.tsx` — коллизия).
- **`findGeometryViolations` пропускает пары одной колонки** (`same cargoTypeId+x+y`) — для ручных
  правок нужна отдельная проверка футпринтов (`editLayout.ts`, хелпер `overlapsOtherStack`).
- **Ручной поворот — только yaw** (`lwh`↔`wlh`, ADR 013). Опрокидывание на грань меняет `dz`, а
  значит z всех ярусов стопки, посчитанные под исходную высоту, — это перерасчёт стопки, а не
  ручная правка. Якорь поворота — угол `(x,y)`: центр-сохраняющий дал бы дробные координаты
  (внутри — целые мм, сетка 100 мм).
- **`localStorage.ladungsplaner.palletPresets` переживает «Zurücksetzen»** — это каталог справочника,
  а не черновик. `test-setup.ts` чистит localStorage в `afterEach`, иначе пресеты текут между тестами.
- **vitest без globals** — авто-cleanup `@testing-library` подключён вручную в `apps/web/src/test-setup.ts`
  (там же чистится `localStorage` в afterEach, иначе persistence-состояние течёт между тестами).
- **Chrome НЕ печатает SVG `<pattern>`** (заливка/штриховка пропадают на бумаге, в т.ч. ч/б). Штриховка
  заказов рисуется прямыми `<line>`/`<circle>` (`HatchMarks`, аналитический клиппинг), как сетка. Не
  возвращать `<pattern>` для печатаемого контента.
- **Persistence-ключи `localStorage`:** `ladungsplaner.setup` (форма), `ladungsplaner.load` (план),
  `ladungsplaner.locale` (язык). Это кэш рабочего черновика; будущий импорт ERPNext (s17) перезапишет,
  «Zurücksetzen» очищает (память `ladungsplaner-persistence-erpnext-2026-07-15`).
- **click-outside слушать `click`, НЕ `mousedown`** — иначе закрытие панели сдвигает DOM до `mouseup`,
  браузер шлёт click на предка и кнопка («+ Position») не срабатывает (`LKWkalk-4bj.1`).
- **Конвенция вида сбоку:** наблюдатель у нижнего края вида сверху (`y = width`) → больший `y` = ближний
  ряд (`depth 0`), меньший — дальше (тусклее). Поле `CutRect.rowY` (`LKWkalk-4bj.2`).
- **Длинные RU-лейблы ломают вёрстку** — текстовые контролы ограничивать по ширине + `truncate`;
  строка позиции `xl:flex-nowrap`, ниже — перенос.
- **jsdom без layout** — баги вёрстки (сдвиг DOM, overflow) тестами не воспроизводятся; тест сторожит
  намерение, а не симптом.
- **Docker-демон может быть выключен** — локальный docker-гейт не отработает; сборку образа фактически
  проверит успешный деплой Coolify.
- **ERPNext токен** ранее давал 403 на Sales Order Item — при включении интеграции выдать доступ.

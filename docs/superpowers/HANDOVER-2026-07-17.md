# HANDOVER — Ladungsplaner (состояние на 2026-07-17, вечер)

> Полный снимок проекта для продолжения в новой сессии/вкладке. Аудитория: следующий агент Claude Code
> + диспетчер claude.ai (читает GitHub). Живое приложение, вся работа в этом монорепо.
> Предыдущие снимки: [HANDOVER-2026-07-16.md](HANDOVER-2026-07-16.md) (архитектура §2–6, не дублирую).

## 1. TL;DR

- **Продукт:** Ladungsplaner — расчёт загрузки грузовика паллетами (3D bin packing, модель 2.5D).
- **LIVE:** **https://ladungsplaner.holz-schaefer.de** (Coolify/Hetzner, TLS, авто-деплой с ветки `production`).
- **Гейты:** **365 тестов**, lint, typecheck (6 воркспейсов), web build — зелёные.
- **Контракт движка — `0.12.0`** (в этой сессии вырос с 0.11.0: операции ручной правки, ADR 019).
  Пакет `@shadrin-v/engine` бампнут до **0.0.7**, **НЕ опубликован** — публикует владелец
  (`docs/onboarding.md` §6), когда понадобится.
- **Эта сессия (2026-07-17, вечер):** экспорт PDF/PNG/JSON (`qrd.15`); UX-батч одностраничности
  (эпик `rgv`, 7 задач); QA-раунд 2 (подпись демо, ПЕРЁД/ЗАД, переименование режима); **буфер стопок**
  (эпик `dwc`: ядро правок + рабочая полоса); разбор плотности «Сбоку» (`u02`) — диагноз без кода.
  Всё на проде и проверено на живом сайте. Детали — §3.

## 3. Что сделано в этой сессии

### `qrd.15` — экспорт PDF / PNG / JSON (закрыт)

Группа «Экспорт» на Ladeplan. Спека `specs/2026-07-17-export-design.md`.
- **PDF** = системный диалог печати («Als PDF speichern»): лист A4 уже вылизан, jsPDF/svg2pdf
  (~400 КБ) дал бы второй движок вёрстки и новый набор багов. У кнопки — `InfoHint`.
- **PNG** = `lib/exportPlan.ts`: заголовок + мета + сводка + **обе проекции** + легенда собираются в
  один standalone SVG → `resolveCssVars` → `<canvas>` 2400 px.
- **JSON** = `{ exportedAt, engineContractVersion, load, layout }` по контракту.
- Экспортируется **отредактированная** раскладка. Новых зависимостей нет.

### Эпик `rgv` — UX одностраничности (7 задач закрыты, `rgv.8` открыт)

Спека `specs/2026-07-17-one-page-ux-design.md`. Решения владельца: **заглушка до расчёта** (авто-пересчёт
отклонён — ручные правки не должны пропадать от правки поля) и **3 демо-варианта по фиксированному кругу**.
- Убрана кнопка «Назад» (рудимент v1: удаляла план, дублируя «Сбросить»).
- Секция плана всегда на странице (`EmptyPlan` до первого расчёта, `print:hidden`).
- Панель действий → именованные группы «Режим загрузки» | «Экспорт» + «Печать».
- «Демо» переехало в шапку «Заказы»; карусель: смешанные заказы · вложение и штабели · перегруз.
- Заголовки колонок у позиций (от `xl`), «Не размещено N» красным в сводке.

### QA-раунд 2 (закрыт)

Подпись демо: сначала состав, потом «нажмите ещё раз». **ПЕРЁД/ЗАД** — под видом сверху, внутри его
фигуры (обе проекции в одной оси X → одна пара маркеров). **«Автоматически» → «Сзади и сбоку»**
(DE «Hinten und Seite») + подсказка: считаются оба варианта, показывается плотнейший.

### Эпик `dwc` — буфер стопок (`dwc.1`–`.4` закрыты, `dwc.5` открыт)

Боль владельца: *«если места не хватает — не даёт повернуть, а неразмещённые неоткуда взять»*.

- **[ADR 019](../adr/019-manual-layout-editing-api.md), контракт 0.12.0 — алгебра правок переехала в
  ядро.** `moveStack`/`rotateStack` **перенесены** из `apps/web` (`editLayout.ts`: 121 → 31 строка,
  остался только snap); добавлены `unplaceStack`, `placeStack`, `stackBuffer`
  (`packages/engine/src/packing/edit.ts`).
- **`EditResult = { layout, error? }`** — при отказе ИСХОДНЫЙ layout + код `ERR_EDIT_*`
  (NO_STACK/OVERLAP/OUT_OF_BOUNDS/FORK_ACCESS/ROTATION/NOTHING_TO_PLACE), переводы в i18n.
  Причины упорядочены: **сначала габарит кузова, потом пересечение**.
- **UI:** `BufferStrip.tsx` — полоса ПОД видом сверху (боковая панель отвергнута: кузов 13,6 м).
  Плитки = **стопки**, не штуки. Двусторонний drag: из кузова на полосу → `unplace`; из полосы в
  кузов → `place` (курсор держит стопку за середину, угол прижимается к габариту). Поворот в буфере.
- Проверено в Chrome: круг «вынуть → буфер 31 шт. → поставить в свободный хвост → буфер пуст», 0 нарушений.

### `u02` — плотность режима «Сбоку»: диагноз, кода нет (закрыт)

Замер: `rear` 704 поддона / 92.9 % пола, `side` 600 / 71.1 % (`strict` 542 / 56.6 %), нарушений 0.
**Раскладка корректна** — это потолок жадной shelf-модели: в `side` полки растут поперёк ширины
(2430) → влезает ровно две, глубину задаёт самый толстый тип (2×1200 из 2430), мелкое дарит разницу
в карман; остаток (EPAL 3, `rotation:'none'`, нужна глубина 1200) в карманы 600 не лезет.
**Замерено и отвергнуто:** политика «мелкая полка» (форсированная — хуже: 542 против 600); поворот
остатка в кармане (0 прироста); полки по классам глубины (−198 поддонов). Код откачен.
[ADR 012](../adr/012-loading-modes.md) дополнен поправкой: «ёмкость симметрична» верно **только для
однородного** груза. Настоящее лечение — 2D max-rects → **`LKWkalk-i1p`**.

## 5. Подводные камни (новое этой сессии)

**Экспорт (`qrd.15`):**
- Разрезы искать **только по `data-cutaway`**: селектор `svg[role="img"]` цепляет свотчи легенды и
  StackDiagram → PNG высотой 7697 px.
- В `<img>` из blob **`var(--…)` не резолвится** → без `resolveCssVars` картинка была бы чёрной.
- `downloadBlob` отзывает objectURL через `setTimeout 0`, не инлайн (Safari/Firefox отменяют закачку).
- Цифры сводки — единый массив `figures` (мета-полоса и PNG из него же).

**Буфер (`dwc`):**
- Ориентация плитки — **по типу груза**, не по индексу списка (список пересобирается после каждой
  правки → индексная ориентация достаётся случайной стопке).
- Drag начинается в HTML и кончается в SVG → глобальные `window` pointermove/up + `getScreenCTM`
  целевого `svg[data-cutaway="top"]`. Выход стопки ИЗ кузова ловится через pointer capture
  (`onUp` приходит в svg, даже когда курсор снаружи) → `onDropOutside`.
- `select-none` на полосе и на `figure` разрезов — иначе drag размазывает выделение текста.
- `applyEdit` **не** вызывает setState внутри updater'а (updater обязан быть чистым); `editError`
  гасится в эффекте сброса `edited`.

**Вёрстка (`rgv.6`):** заголовки колонок выравниваются только при **фиксированных** ширинах —
`flex-1` у имени растягивался в шапке иначе, чем в строке (сдвиг ~34 px). Ширины: свотч `w-3`,
артикул `w-36`, имя `w-32`, Д/Ш/В `w-24`, кол-во `w-20`. Строка тратит ~1026 из 1036 px — более
широкое имя выбрасывает чип «Stapel N» на вторую строку.

**Проверка в браузере (обязательна, jsdom без вёрстки и без canvas):** `puppeteer-core` +
`/Applications/Google Chrome.app`; **`scrollIntoView`** перед drag — буфер ниже окна, мышь туда не
доедет; для скачиваний — CDP `Browser.setDownloadBehavior`.

**Деплой:** сразу после смены имени бандла возможна **гонка** — `index.html` уже новый, а
`/assets/index-*.js` ещё 404 (отдаётся SPA-фолбэк, ~460 байт). Подождать ~20 с и перекачать.

**Остаются в силе** (из [HANDOVER-2026-07-16.md](HANDOVER-2026-07-16.md) §9 и утренней части):
Chrome не печатает SVG `<pattern>` (штриховка прямыми `<line>`); click-outside на `click`, не
`mousedown`; конвенция вида сбоку (`rowY`, больший `y` = ближе); `findGeometryViolations` пропускает
пары одной колонки; ручной поворот только yaw; `dist` в .gitignore (пересобирать engine/i18n перед
web-тестами); `densityFirst ≥ strict`; эталоны упаковщика **EUR 34 / GB 20 / none 33** —
регресс-сторож; **git checkout+merge+push по отдельности** (не через `&&`).

## 2–6. Архитектура / экраны / ERPNext / деплой / процесс

**Без изменений** относительно [HANDOVER-2026-07-16.md](HANDOVER-2026-07-16.md) §2–6, кроме уточнений ниже.
Кратко: монорепо `packages/{engine,i18n,contracts}` + `apps/{web,server}`; движок 2.5D работает **в браузере**
(ADR 001); шов `DataProvider`; персистенс SQLite; i18n de/ru ключами; дизайн-токены (ни одного hex в JSX);
инвариант `findGeometryViolations(load, layout) === []` на каждом результате. Автодеплой: push в `production`
(= `main`) → Coolify webhook → build → live.

**Уточнение по проверке деплоя (важно, добыто в этой сессии):** сборка Coolify может дать **другой content-hash**
бандла, чем локальная `npm run build` (разное окружение: node/esbuild). Поэтому матчить прод по ЛОКАЛЬНОМУ
имени `index-*.js` **ненадёжно**. Надёжный сигнал выката: (1) `git ls-remote origin production` == твой HEAD И
(2) имя `/assets/index-*.js` на проде **сменилось** относительно прошлого деплоя. Не жди совпадения с локальным хешем.

- ✅ **`ymv`** — карманы в shelf-упаковщике → второй проход **`backfillPockets`** (`floor.ts`): собирает

  гильотинным best-fit **той же ориентацией** → строго-добавляющий, эталоны (34/20/33)/геометрия/детерминизм

  кармане замерен (0 прироста на демо) — не внедрён (YAGNI). ADR 017 дополнен.

## 9. Подводные камни (новое этой сессии + актуализация)

**Новое (2026-07-17):**
- **densityFirst ОБЯЗАН быть ≥ strict** (инвариант ADR 016). `packLoad` считает обе раскладки и берёт
  плотнейшую. Не «упрощай» обратно в один регион — вернётся баг `r0m`.
- **Демо насыщено (~93% пола)** — видимые «свободные места» в демо неустранимы упаковщиком (габариты не
  бьются на мелкие куски). Не гоняйся за плотностью на демо; замеряй на ненасыщенном грузе.
- **`backfillPockets` строго-добавляющий и той же ориентацией** — поэтому эталоны 34/20/33 держатся. Если
  захочешь вращать в кармане (плотнее) — проверь, что эталоны не поехали (могут: напр. Gitterbox 20→21).
- **Цвет заказа = стабильный `colorIndex`, НЕ позиция в списке.** Красит `orderColorToken(o.colorIndex)`.
  Карта `orderId→colorIndex` персистится в `ladungsplaner.orderColors` и прокидывается в Ladeplan
  (`cutaway.ts`/`orderBreakdown.ts` берут её, fallback — appearance order). Порядок легенды — по появлению.
- **Демо транзиентно:** `skipNextSaveRef` в `SetupScreen` + `result.transient` в `App`. Не персистить демо
  (ни setup, ни load). Смена стратегии на демо-плане тоже сохраняет транзиентность (`persist:!result.transient`).
- **Проверка деплоя — по production-ref + смене хеша, НЕ по совпадению с локальным хешем** (Coolify собирает
  в другом окружении, хеш может отличаться). См. §2–6 «Уточнение».
- **Новый localStorage-ключ `ladungsplaner.orderColors`** (карта цветов заказов; чистится вместе с планом).
- **Кнопка вращения стопки — ВНУТРИ фигуры** (иначе обрезается у краёв кузова, `rrx`).
- **`InfoHint` умеет `align='right'`** (открыть тултип влево) — для контролов у правого края.

**Актуализация старых камней:**
- ⚠️ **Строка позиции теперь `flex-wrap` (БЕЗ `xl:flex-nowrap`)** — старый камень про nowrap устарел. Длинные
  имена/доп.контролы переносятся на вторую строку, имя ограничено `max-w-[14rem]`.

**Остаются в силе** (из [HANDOVER-2026-07-16.md](HANDOVER-2026-07-16.md) §9): Chrome не печатает SVG `<pattern>`
(штриховка прямыми `<line>`); click-outside на `click` не `mousedown`; конвенция вида сбоку (`rowY`, больший
`y`=ближе); `findGeometryViolations` пропускает пары одной колонки (для ручных правок — `overlapsOtherStack`);
ручной поворот только yaw, якорь — угол; `dist` в .gitignore (пересобирать engine/i18n перед web-тестами);
jsdom без layout; InfoHint (`<button>`) держать ВНЕ `<label>`; upacker best-fit по открытым полкам + backfill,
эталоны EUR34/GB20/none33 — регресс-сторож; **git checkout+merge+push по отдельности** (не `&&`, хук
авто-экспортит `.beads/issues.jsonl` при commit).

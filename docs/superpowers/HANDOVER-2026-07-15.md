## 1. TL;DR

- **Продукт:** Ladungsplaner — расчёт загрузки грузовика паллетами (3D bin packing, модель 2.5D).
- **LIVE:** **https://ladungsplaner.holz-schaefer.de** (Coolify/Hetzner, TLS). Полный флоу работает.
- **Гейты:** 214 тестов, lint, typecheck (6 воркспейсов), docker build — всё зелёное.
- **Состояние:** MVP-приложение готово и развёрнуто; идут итеративные доработки UX. Открыты только
  owner-side follow-up (ниже §7).

## 2. Архитектура

Монорепо (npm workspaces `packages/*` + `apps/*`):
```
packages/engine    @shadrin-v/engine  — headless движок 2.5D (контракт 0.9.0). Работает В БРАУЗЕРЕ.
packages/i18n      @shadrin-v/i18n     — локали de/ru, форматирование. Строки — только ключи.
packages/contracts @shadrin-v/contracts— DTO API (граница web↔server).
apps/web           Vite React SPA      — два экрана (Настройка, Ladeplan), одна страница.
apps/server        Fastify + better-sqlite3 — персистенс + адаптер ERPNext + отдаёт SPA.
```
- **Движок в браузере** (ADR 001): SPA сама зовёт `calculateLayout`, `computeStack`, `orientedDims`,
  `findGeometryViolations`. Сервер тонкий.
- **Шов `DataProvider`** (`apps/web/src/data/`): единственный интерфейс данных. Сейчас `HttpDataProvider`
  → `apps/server`. B→A: замена реализации на Frappe без правки экранов (ADR 015).
- **Персистенс:** SQLite (`apps/server`, том `/app/data`), план = снимок `Load`+`Layout`+ref заказов.
- **i18n с первого коммита** (ADR 006), **токены дизайна** (design-system) — ни одного hex в JSX.
- **Геометро-инвариант:** `findGeometryViolations(load, layout) === []` на каждом результате.

## 3. Экраны (apps/web)

- **Настройка** (`screens/SetupScreen.tsx`): строка кузова + пресеты, карточки заказов (зоны), компактные
  строки-позиции — пресет-палета, размеры, кол-во, segmented Ent/Ver, вращение, live-чип штабеля,
  раскрытие правил (режим вложения sequential/pairwise, Δh/h_д, maxNested, allowUnpairedTop, maxTiers),
  **живая формула** штабеля, блок «Berechnen» при некорректном Δh. Строит `Load` по зонам-заказам.
- **Ladeplan** (`screens/LadeplanScreen.tsx`): два разреза SVG в мм (`components/CrossSection.tsx` +
  `components/cutaway.ts`), легенда (цвет+штрих), метрики, печать A4, **ручной drag** штабелей в виде
  сверху (`components/dragLayout.ts`, snap 100мм + ревалидация). Рендерит редактируемую копию раскладки.
- **Одна страница** (`App.tsx`): SetupScreen всегда смонтирован, Ladeplan ниже при результате → состояние
  не сбрасывается. Переключатель языка **DE|RU** (`ui/LocaleSwitch.tsx`, localStorage).
- Дизайн-система: **Direction D**, бренд Holz Schäfer (`docs/design/design-system.md` + `theme.css` →
  `apps/web/src/theme.css` + `tailwind.config.js`). Токены: brand `#104F25`, mint `#71BF87`, sage `#C3D1C7`.

## 4. ERPNext (интеграция, вариант B)

- Адаптер `apps/server/src/erpnext/adapter.ts`: `importOrder` (Sales Order → зона), `searchOrders`,
  token-auth. Габариты — из **кастом-полей** `custom_length_mm/width/height` строки Sales Order
  (`fetch_from` Item); **парсера из названия нет** (по спеке). Провенанс `erpnext-field` / `manual`.
- REST `/api/orders`, `/api/orders/:id` с guard **`ERR_ERPNEXT_UNCONFIGURED` (503)**, когда секреты не
  заданы. **Сейчас ERPNext в локальном тест-режиме** — секреты не заданы, `/api/orders` отдаёт 503, это
  нормально. Поля в ERPNext создаст ERP-агент (задача `LKWkalk-k06`, спека
  `docs/superpowers/specs/2026-07-14-erpnext-fields-task-for-erp-agent.md`).

## 5. Деплой (Coolify / Hetzner)

- **Рунбук:** `docs/INFRASTRUKTUR-ladungsplaner.md`. Сервер Hetzner CX23, Coolify 4.1.2, Traefik+TLS.
- **Ветка автодеплоя:** `production` (= `main`). Dockerfile multi-stage (Node 22, better-sqlite3 native).
- **Env:** `PORT=3000`, `STATIC_DIR=/app/web`, `DB_PATH=/app/data/app.db`. ERPNext-секреты — не заданы.
- **Том:** `/app/data` (SQLite). Лимиты 512M/0.5CPU. Домен `ladungsplaner.holz-schaefer.de` (зона
  holz-schaefer.de; сервер/панель Coolify — на group-schaefer.de).
- ⚠️ **Auto-deploy:** источник deploy-key НЕ деплоит на push автоматически → нужен ручной **Redeploy**
  в панели. Задача `LKWkalk-la7` — настроить GitHub-webhook, чтобы это стало автоматом.
- **Проверка живого бандла:** скачать `/assets/index-*.js` в файл и `grep` (НЕ в shell-переменную —
  170КБ манглится). Vite хэширует по содержимому → совпадение имени = байт-в-байт та же сборка.

## 6. Как вести работу (рабочий процесс)

- **Docs-first:** изменилось требование/API — сначала spec/ADR/contract, потом код.
- **Процесс:** фича → `superpowers:brainstorming` → спека (`docs/superpowers/specs/`) → `writing-plans`
  (`docs/superpowers/plans/`) → TDD. Баг → `systematic-debugging`.
- **Автономия (память `autonomy-directive-2026-07-10`):** beads team-maintainer — авто commit/push/
  `bd dolt push`/close после зелёных гейтов; **ветка-на-задачу → merge в main сам**.
- **Гейты (перед merge):** `npm test` · `npm run lint` · `npm run typecheck` · `npm run build --workspace apps/web`
  · `docker build` (при изменении web/server). При правке `packages/engine` или `packages/i18n` — пересобрать
  их dist (`npm run build --workspace @shadrin-v/<pkg>`), т.к. web/server импортируют dist (dist в .gitignore).
- **Beads:** `bd ready` / `bd show <id>` / `bd update --status` / `bd close`. Эпик = группировка (НЕ вешать
  blocking-dep задача→эпик, иначе задачи блокируются).
- **Верификация live:** после merge — push `main` + `production`; затем Redeploy (пока нет webhook) →
  проверить бандл на проде.

## 9. Известные подводные камни

- **Coolify не авто-деплоит** (до webhook `la7`) — Redeploy вручную.
- **dist в .gitignore** — на чистой машине сперва `npm ci` + сборка пакетов, иначе web/server-тесты не
  видят `@shadrin-v/engine`/`i18n`.
- **macOS case-insensitive FS** — не создавать в одной папке файлы, различающиеся только регистром
  (`CrossSection.tsx` vs `cutaway.ts` — ок; `crossSection.ts` vs `CrossSection.tsx` — коллизия).
- **`findGeometryViolations` пропускает пары одной колонки** (`same cargoTypeId+x+y`) — для drag нужна
  отдельная проверка футпринтов (`dragLayout.ts`).
- **vitest без globals** — авто-cleanup `@testing-library` подключён вручную в `apps/web/src/test-setup.ts`.
- **ERPNext токен** ранее давал 403 на Sales Order Item — при включении интеграции выдать доступ.

   → merge в main сам**, docs-first, TDD, геометро-валидатор на каждом результате. Спрашивать ТОЛЬКО про
   ломающий контракт или продуктовое решение без очевидного дефолта (сразу с предложенным дефолтом).
2. Прочитай **`docs/superpowers/HANDOVER-2026-07-17.md`** — свежий снимок (что сделано, статус,

**Процесс (не перескакивай):**
- Новая фича → короткий дизайн-раздел/спека в `docs/superpowers/specs/` → `writing-plans` → TDD.
- Баг → `superpowers:systematic-debugging` (гипотеза + воспроизведение с замером, потом правка).
- Перед merge: `npm test` · `npm run lint` · `npm run typecheck` · `npm run build --workspace apps/web`.
  Правил `packages/engine`/`i18n` → **пересобери их dist** (web/server импортируют dist, он в .gitignore).
- После merge в `main` → Coolify авто-деплоит (continuous deployment, ADR 023). Отдельной ветки `production` нет.
  **Проверка выката:** `git ls-remote origin production` == HEAD И имя `/assets/index-*.js` на проде
  **сменилось** (хеш Coolify может НЕ совпасть с локальным). Сразу после смены имени возможна гонка:
  ассет ещё 404 и отдаётся SPA-фолбэк (~460 байт) — подождать ~20 с и перекачать.
- **Проверка в реальном Chrome обязательна** для вёрстки, drag и canvas (jsdom не умеет ни то, ни другое):
  `puppeteer-core` + `/Applications/Google Chrome.app`. Перед drag — `scrollIntoView` (буфер ниже окна).

MVP развёрнут и работает. Закрыты эпики: `qrd` (движок MVP), `66g` (full-stack), `563` (дизайн),
`2ll`/`fvx` (web UX), `4bj` (Ladeplan v2), `rgv` (одностраничность), `dwc` (буфер стопок).

Последняя сессия дала: **экспорт PDF/PNG/JSON**; **одностраничность** (план всегда на экране, заглушка
до расчёта, именованные группы панели, демо-карусель из 3 вариантов, заголовки колонок, «не размещено»

отказами; **разбор `u02`** (плотность «Сбоку» — потолок модели, не баг). Полный статус — HANDOVER §3.

**Ключевое архитектурное изменение:** алгебра ручных правок живёт **в ядре** ([ADR 019](../adr/019-manual-layout-editing-api.md),
контракт 0.12.0): `moveStack`/`rotateStack`/`unplaceStack`/`placeStack`/`stackBuffer` возвращают

держит только snap-сетку. Метрики и построение колонки — только ядро, никогда не считай их в UI.

⚠️ Пакет `@shadrin-v/engine` бампнут до **0.0.7** (контракт 0.12.0) и **НЕ опубликован** — публикует

## Задача: выбрать из бэклога (`bd ready`)

- **`qrd.31`** — зазор (clearance) на уровне позиции груза (`CargoType.clearance`) вместо `Load.clearance`.
  Аддитивно к контракту; был следующим кандидатом.
- **`qrd.29`** — дефолт `nestingMode` = pairwise. **Спроси владельца**: меняет видимое поведение расчёта.
- **`rgv.8`** — дубль «пресет + название» в строке позиции. **Нужно решение владельца по форме.**
  Предложение агента: название автозаполняется из пресета и уходит в раскрывающуюся часть.
- **`dwc.5`** — постановка стопки из буфера с клавиатуры (сейчас только указателем; повернуть с
  клавиатуры можно, а поставить нет — несимметрично).
- **`qrd.21`** — интеграционный тест: full-груз tipped-only проходит валидацию, но unplaced (ADR 013).
- **`i1p`** — настоящий 2D-упаковщик (max-rects) вместо полок. Большой; риск для эталонов; ценность
  ограничена (дефолт «Сзади и сбоку» уже отдаёт плотнейший проход).
- **`qrd.17`** — реальные пресеты (нужны данные владельца). **Owner-side:** `i6b` (**Basic Auth — прод
  сейчас БЕЗ авторизации**), `zbi` (бэкап тома), `s17`/`k06` (ERPNext).

**Инварианты (держать всегда):** `findGeometryViolations(load, layout) === []` на каждом результате;
правила nesting/stacking/rotation/forkAccess не нарушаются ради заполнения; расчёт детерминирован;
`densityFirst ≥ strict`; **`placed + unplaced` по типу сохраняется при ручных правках**; ручная стопка
не выше расчётной (строится тем же `columnPlacements`); эталоны упаковщика **EUR 34 / GB 20 / none 33**.

## Подводные камни, добытые кровью (не наступай)

- **Экспорт PNG:** разрезы искать **только по `data-cutaway`** (`svg[role="img"]` цепляет свотчи легенды
  → PNG 7697 px); `var(--…)` в `<img>` из blob не резолвится → `resolveCssVars` обязателен;
  `downloadBlob` отзывает objectURL через `setTimeout 0` (иначе Safari/Firefox рвут закачку).
- **Буфер:** ориентация плитки — **по типу груза**, не по индексу списка (список пересобирается после
  каждой правки); drag HTML→SVG живёт на глобальных `window`-слушателях + `getScreenCTM`; выход стопки
  из кузова ловится pointer capture → `onDropOutside`; `select-none` на полосе и `figure` разрезов.
- **React:** не вызывай setState внутри updater'а другого setState (`applyEdit` считает правку снаружи).
- **Вёрстка строки позиции:** ширины колонок **фиксированы** (`flex-1` у имени ломал выравнивание
  заголовков на ~34 px); строка тратит ~1026 из 1036 px — шире имя → чип «Stapel N» уезжает на 2-ю строку.
- **Демо насыщено** (~93 % пола в варианте 1) — не гоняйся за плотностью на нём; замеряй на ненасыщенном.
- **`side` беднее `rear`** (600 против 704 на демо) — это потолок shelf-модели, НЕ баг: см. поправку в
  [ADR 012](../adr/012-loading-modes.md). Дешёвые заплатки замерены и отвергнуты (`u02`), не повторяй их.
- **Chrome НЕ печатает SVG `<pattern>`** — штриховка прямыми `<line>`/`<circle>` (`HatchMarks`).
- **click-outside слушать `click`, НЕ `mousedown`**; **ручной поворот — только yaw**; **вид сбоку:**
  больший `y` = ближний ряд; **`findGeometryViolations` пропускает пары одной колонки**.
- **`localStorage`-ключи:** `ladungsplaner.setup` · `.load` · `.orderColors` · `.locale` · `.palletPresets`
  (последний переживает Reset). Демо **транзиентно** (`skipNextSaveRef` + `result.transient`).
- **`dist` в .gitignore** — на чистой машине `npm ci` + сборка пакетов, иначе web/server-тесты не видят
  `@shadrin-v/engine`/`i18n`. **git checkout+merge+push — по отдельности, НЕ через `&&`** (хук
  авто-экспортит `.beads/issues.jsonl` при commit).

**Что делаем сейчас:** у владельца есть правки — спроси/дождись их. Если правок нет — бери из бэклога
по приоритету (`qrd.31` — следующий кандидат). Работаем по одной: ветка → зелёные гейты → merge в main
→ мерж в main → Coolify авто-деплоит → проверка на проде.

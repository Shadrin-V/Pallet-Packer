# ADR 011 — Группировка по заказам (без LIFO-очередности)

Статус: Принято · Дата: 2026-07-09

## Контекст

Смешанные загрузки — частый кейс (уточнено пользователем; отменяет «mixed редкий» из
[ADR 005](005-order-fulfillment-mode.md)). Практическое требование: груз одного заказа/клиента
должен выгружаться целиком, не разбирая чужие поддоны. Пример: заказ1 = A+B клиенту1,
заказ2 = A+C клиенту2 (один тип поддона может быть в разных заказах).

Полная выгрузка по маршруту (LIFO-очередность точек) в MVP не требуется — нужна только
**группировка** (пользователь выбрал вариант «только группировка»).

## Решение

- Позиции груза тегируются `CargoType.orderId?: string` (аддитивно, контракт 0.3.0). Позиции с
  одинаковым `orderId` образуют один заказ. Без `orderId` — единая неявная группа.
- Упаковщик размещает поддоны одного заказа **смежной зоной** (не перемешивая с другими заказами),
  чтобы заказ выгружался целиком.
- **Порядок зон между собой — не строгий** (LIFO-последовательность точек и сторона двери в MVP
  не реализуются; заложено на будущее).
- Ориентация футпринта выбирается по **макс-влезанию** (не фиксирована), в т.ч. внутри зоны заказа.

## Последствия

- Упаковщик становится по-настоящему гетерогенным (shelf/skyline), с зонированием по `orderId`
  ([ADR 004](004-packer-interface-shelf-heuristic.md) обновлён).
- Затрагивает floor-упаковщик (`LKWkalk-qrd.4`) и оркестратор (`LKWkalk-qrd.7`).
- LIFO-очередность выгрузки по точкам маршрута остаётся вне MVP (только группировка зон).
- Возможен компромисс утилизации: смежность зоны заказа может оставлять пустоты — приемлемо ради
  выгрузки целиком.

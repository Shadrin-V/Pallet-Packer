# ADR 002 — Внутреннее представление размеров: целые миллиметры

Статус: Принято · Дата: 2026-07-09

## Контекст

Геометрические инварианты (нет пересечений, нет выхода за габариты) и property-based тесты
требуют точных сравнений. Плавающая точка вносит погрешности и требует epsilon-сравнений.

## Решение

Внутри ядра все линейные размеры и координаты — **целые числа в миллиметрах**. Ввод/вывод в UI
может быть в мм/см/м; конвертация выполняется только на границе (UI/i18n). Стандартные размеры
поддонов и кузовов естественно выражаются в целых мм (EUR 1200×800).

## Последствия

- (+) Детерминированная геометрия, точные сравнения, надёжные инварианты и тесты.
- (+) Нет дрейфа погрешности при сложении координат.
- (−) Дробные единицы (напр. 0.5 мм) не поддерживаются внутри — для логистики несущественно.
- Конвертация единиц — ответственность слоя i18n/UI, не ядра.

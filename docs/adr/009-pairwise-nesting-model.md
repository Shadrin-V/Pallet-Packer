# ADR 009 — Модель вложения: pairwise (парами)

Статус: Принято · Дата: 2026-07-09
(Правило обрезки по `maxNested` — предварительно, ждёт подтверждения, см. «Открытый вопрос».)

## Контекст

Физика вложения уточнена пользователем. Ранее ([ADR 003](003-2p5d-computation-model.md)) вложение
считалось последовательным: каждая вложенная единица добавляет Δh. Для реальных каркасов вложение
идёт **парами**: второй поддон переворачивается и входит в первый «замком», образуя пару; пары
ставятся друг на друга; нижний поддон стопки всегда одиночный.

## Решение

Режим вложения задаётся на уровне типа (аддитивно к контракту, минорная 0.2.0):

- `nestingMode: 'sequential' | 'pairwise'` (default `sequential` — прежнее поведение сохраняется).
- Для `pairwise` поле `stepHeight` интерпретируется как **h_д** — высота двух верхних досок
  перевёрнутого поддона (прирост высоты пары сверх H).
- `allowUnpairedTop?: boolean` (default `false`) — класть одиночный непарный поддон сверху,
  если остаток высоты ≥ H.

### Формула pairwise
```
k = floor((Hк − H) / (H + h_д))                              // число пар
n = 2k + 1                                                   // 1 нижний одиночный + k пар
если allowUnpairedTop и (Hк − (H + k·(H + h_д))) ≥ H → n += 1 // непарный сверху
n = min(n, maxNested)                                        // обрезка
```
Структура стопки при итоговом n: 1 нижний одиночный + пары + [непарный сверху].
Инвариант высоты: `H + k·(H + h_д) [+ H] ≤ Hк`.

### Эталонные значения
- H=144, h_д=22, Hк=2650 → k=15, n=31, высота 2634 (остаток 16, флаг не влияет).
- H=144, h_д=22, Hк=2790, `allowUnpairedTop=true` → n=32 (остаток 156 ≥ 144); `false` → n=31.
- `maxNested=6` на «1+3 пары+непарный» → «1+2 пары+непарный», n=6 (пары не разрываются).

## Последствия

- Контракт минорно растёт до 0.2.0 (только добавления); `sequential` не меняется.
- Вертикальный расчёт (`LKWkalk-qrd.5`) реализует обе формулы по `nestingMode`.
- UI (`LKWkalk-qrd.13`): выбор режима, поле h_д (de: «Höhe der oberen Bretter»), чекбокс
  «одиночный поддон сверху».
- Вид сбоку (`LKWkalk-qrd.14`) визуально различает пары и одиночный верхний.
- Инвариант [spec.md](../spec.md) §11 переформулирован под активный `nestingMode`.

## Открытый вопрос (ждёт подтверждения)

Правило обрезки по `maxNested`. Прозаичное описание («сверху: сначала непарный, затем целыми
парами») при буквальном чтении для эталона `maxNested=6` даёт n=5 (убрать непарный → 7, убрать
пару → 5), тогда как эталонный результат — n=6 («1+2 пары+непарный»). Здесь зафиксирован
тест-совместимый вариант: `n = min(n_full, maxNested)`, затем структура выводится из n
(1 нижний + `floor((n−1)/2)` пар + непарный, если `(n−1)` нечётно и `allowUnpairedTop`).
Требуется подтверждение либо уточнение.

# ADR 015 — Полноценное приложение + интеграция ERPNext (вариант B → A)

Статус: Принято · Дата: 2026-07-14

## Контекст

UI-прототип в Lovable отвергнут — Lovable не воспроизводит утверждённый дизайн. Дальнейшая разработка
идёт через Claude Code. Требуется **полноценное приложение с БД** и **полной интеграцией с ERPNext**
(облако Schäfer): товары ERPNext — сами паллеты; поток «в один клик» из Sales Order создаёт заказ с
позициями и габаритами, при неполной машине добавляется следующий заказ. Есть готовая инфраструктура
(Hetzner + Coolify + Docker + SQLite, паттерн приложения `arminia`) и готовый движок
`@shadrin-v/engine`.

## Решение

Строим приложение **вариантом B → A**:

- **B (сейчас):** отдельное приложение, интеграция с ERPNext по REST. Стек (утверждён, «вариант 1»):
  **Vite React SPA + тонкий Fastify-бэкенд (better-sqlite3), один Docker-контейнер**, деплой через
  Coolify (ветка `production`, Traefik+TLS, поддомен `ladungsplaner.group-schaefer.de`, named volume,
  лимиты ~512 MB/0.5 CPU, без публикации портов).
- **A (позже):** то же приложение как **Frappe-app внутри ERPNext**.

Монорепо: переиспользуем `packages/engine` и `packages/i18n` как workspace-зависимости; добавляем
`apps/web` (два экрана из эталонов) и `apps/server`. Движок работает в браузере. Весь доступ к данным —
через интерфейс **`DataProvider`** (в B поверх SQLite+ERPNext REST, в A поверх Frappe), чтобы миграция
B→A была локализованной. Детали — [дизайн-спека](../superpowers/specs/2026-07-14-fullstack-app-erpnext-design.md).

## Последствия

- **ADR 001** (движок в браузере) — сохраняется. **ADR 007** (IndexedDB) — заменён: персистенс в
  SQLite на сервере. **ADR 010** (npm-публикация для Lovable) — неактуален (движок импортируется
  локально; публикация вне критпути).
- UI-биды под Lovable (промпты qrd.13/14/15/23/28/30) — **superseded**: UI строится в `apps/web`.
- Габариты Item в ERPNext пока отсутствуют → фолбэк-парсинг из названия + ручная правка, до добавления
  кастом-полей на стороне ERPNext.
- Аутентификация MVP — Basic Auth за Traefik (или без неё внутри приватного поддомена); ERPNext-SSO — в
  варианте A. Аккаунты/роли/мультитенант/биллинг — вне объёма.
- Новый эпик в beads; прежний MVP-эпик (`qrd`) остаётся источником доменных решений/движка.

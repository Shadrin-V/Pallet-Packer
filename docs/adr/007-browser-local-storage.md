# ADR 007 — Хранение справочников: браузер + пресеты

Статус: Принято · Дата: 2026-07-09

## Контекст

Нужны справочники кузовов и типов поддонов. Движок исполняется в браузере, бэкенда в MVP нет
(см. [ADR 001](001-headless-ts-engine-in-browser.md)). Требуется удобная ежедневная работа и
возможность бэкапа/переноса.

## Решение

Встроенные **пресеты** (типовые фуры; стандартные поддоны EUR 1200×800, FIN 1200×1000, …) плюс
пользовательские записи в **IndexedDB** браузера. Экспорт/импорт всей библиотеки одним JSON-файлом.
CSV/Excel-импорт заявки — вне MVP.

Отклонено на MVP: облачная БД (Supabase) — требует бэкенда и аккаунтов, противоречит browser-MVP;
только JSON-файлы вручную — неудобно в ежедневной работе.

## Последствия

- (+) Ноль бэкенда; «открыл и работаешь»; бэкап/перенос через JSON.
- (−) Данные привязаны к браузеру/устройству; синхронизация между устройствами — будущий эпик
  (тогда появится облачное хранилище и, вероятно, REST-слой).
- Формат JSON-экспорта библиотеки должен быть версионирован для совместимости.

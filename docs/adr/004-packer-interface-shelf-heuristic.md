# ADR 004 — Упаковка за интерфейсом Packer; MVP — shelf-эвристика

Статус: Принято, частично уточнён ([ADR 017](017-dense-floor-heuristic.md)) · Дата: 2026-07-09

> **Уточнения от 2026-07-16 ([ADR 017](017-dense-floor-heuristic.md)):** (1) сорт футпринтов по
> убыванию площади **отменён** — никогда не был реализован, приоритетом остаётся порядок входа;
> (2) next-fit заменён на best-fit + backfill карманов; (3) интерфейс `Packer` в TypeScript **не
> существует** — фактический шов описан ниже.

## Контекст

Точный перебор NP-труден. Нужен детерминированный алгоритм с хорошим балансом скорость/качество,
поддерживающий поворот 90° и разнородные (редко) прямоугольники. При этом важно сохранить
возможность улучшить алгоритм позже без переписывания системы.

## Решение

Упаковка скрыта за **сменяемой стратегией**. В MVP — детерминированный **2D shelf/skyline**
упаковщик: размещение по «полкам» с учётом `clearance` и допустимых поворотов; для одинаковых
поддонов даёт ровные ряды. Вертикаль считается отдельным модулем (см.
[ADR 003](003-2p5d-computation-model.md)).

Фактический шов — чистая функция `packFloor(region, requests, opts) → FloorPlacement[]`
(`packages/engine/src/packing/floor.ts`) с единственным вызовом из оркестратора. Именованного
TypeScript-интерфейса `Packer` нет и он не нужен: сигнатура `packFloor` уже узкая и достаточная —
альтернативная эвристика ставится внутрь неё ([ADR 017](017-dense-floor-heuristic.md)).

~~Сортировка футпринтов по убыванию площади/длинной стороны~~ — **отменена**
([ADR 017](017-dense-floor-heuristic.md)): реализована никогда не была, а порядок входа несёт
семантику **приоритета заявки** ([ADR 005](005-order-fulfillment-mode.md)); плотность достигается
best-fit/backfill без переупорядочивания запросов.

Ориентация футпринта выбирается по **макс-влезанию** (не фиксирована — разворот, если так больше).
Упаковщик **гетерогенный** (смешанные загрузки частые). Позиции одного заказа (`orderId`)
размещаются смежной зоной ([ADR 011](011-order-grouping.md)).

Отклонено на MVP: layer/wall-building (хуже на смешанных), guillotine (теряет место на разнородных),
генетика/метаэвристики (стохастичны, тяжелее — противоречат детерминизму без пользы сейчас).

## Последствия

- (+) Простая, детерминированная, предсказуемая упаковка; легко тестировать.
- (+) Позже можно добавить `PlpPacker` (оптимальные паттерны для одного типа) и метаэвристики
  (со фиксированным seed) без изменения оркестратора и UI.
- (−) Не гарантирует оптимум утилизации; для MVP достаточно.

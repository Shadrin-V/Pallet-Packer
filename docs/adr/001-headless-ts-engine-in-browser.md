# ADR 001 — Headless изоморфный TS-движок, расчёт в браузере

Статус: Принято · Дата: 2026-07-09

## Контекст

Нужен движок упаковки, независимый от UI, с чистым JSON-API. Первый интерфейс строится в Lovable
(генератор React-приложений). Требуется MCP-готовность в будущем. Ключевой вопрос из постановки —
где выполняется расчёт: в браузере или на бэкенде.

## Решение

Движок — **чистый изоморфный TypeScript-пакет** `@pallet/engine` без зависимостей от DOM и Node.
В MVP он **исполняется в браузере**: Lovable импортирует пакет и вызывает `calculateLayout`
локально. Тот же пакет позже оборачивается в Node REST-сервис и MCP-сервер без изменений ядра.

Рассмотренные альтернативы: бэкенд с REST API (отклонено — лишняя инфраструктура для MVP-масштаба);
WASM-ядро на Rust/C++ (отклонено — сложный билд и другой язык, преждевременно).

## Последствия

- (+) Ноль инфраструктуры, офлайн, мгновенный отклик; развилка «браузер/бэкенд» становится
  деталью деплоя, а не архитектуры.
- (+) Один и тот же код ядра переиспользуется в REST и MCP.
- (−) Алгоритм исполняется на клиенте (виден в бандле) — для данного инструмента приемлемо.
- (−) Тяжёлые расчёты ограничены ресурсами браузера; для MVP-масштаба достаточно.

# ADR 006 — i18n de/ru; движок возвращает коды ошибок

Статус: Принято · Дата: 2026-07-09

## Контекст

Контекст эксплуатации немецкий (LKW, Verschachtelt/Entschachtelt), пользователь — русскоязычный.
Требование постановки — i18n с первого дня, ни одной строки в коде. Движок должен оставаться
headless и не зависеть от языка.

## Решение

Локали MVP — **de и ru** (en добавляется позже одним словарём). Все пользовательские строки —
через ключи локалей в `@pallet/i18n`; локализуются интерфейс, подписи отчёта, единицы измерения
и форматы чисел. **Движок текст не хранит** — возвращает коды ошибок (`ERR_*`) и структурные
данные; перевод и форматирование — на стороне UI.

## Последствия

- (+) Ядро остаётся чистым и языконезависимым; те же коды переиспользуют REST/MCP.
- (+) Новые языки добавляются без изменений кода.
- (−) UI обязан сопровождать словари для всех кодов ошибок и ключей.
- (−) Стартовый объём переводов — два языка вместо одного.

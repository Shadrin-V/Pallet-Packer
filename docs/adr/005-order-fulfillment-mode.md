# ADR 005 — Основной режим: «Размести заявку» (список)

Статус: Принято · Дата: 2026-07-09

## Контекст

Возможны два режима: «сколько влезет» (ёмкость для одного типа) и «размести заявку» (список типов
с количествами, показать неразмещённое). Частый случай — один тип; смешанные заявки редки.

## Решение

Основной режим MVP — **«Размести заявку»**: на вход `Load` со списком `cargo: CargoType[]`,
разместить максимум, вернуть `unplaced`. Он общий: одиночный тип = список из одной позиции,
а «сколько влезет» покрывается флагом `fill: true` у позиции. Контракт принимает список всегда,
поэтому смешанные заявки добавляются позже без ломки контракта.

## Последствия

- (+) Один режим покрывает и ёмкость, и заявку; контракт готов к смешанным типам.
- (+) UI-поток единый (редактор заявки).
- (−) Требует от 2D-упаковщика работать с разнородными прямоугольниками (реализуемо shelf-эвристикой).
- Оптимизация смешанных заявок (PLP-паттерны) — вне MVP.

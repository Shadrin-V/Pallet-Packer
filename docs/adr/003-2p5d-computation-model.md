# ADR 003 — Модель расчёта 2.5D: пол + вертикаль вложения

Статус: Принято · Дата: 2026-07-09

## Контекст

Реальная задача — раскладка поддонов при загрузке фуры (аналог назначения «Astra Раскрой»).
Поддоны/каркасы могут находиться в состоянии Verschachtelt (вложены) или Entschachtelt (не вложены),
что меняет число единиц в фуре. Полноценный 3D bin packing произвольных грузов избыточен.

## Решение

Задача моделируется как **2.5D**:

1. **2D-раскладка** футпринтов по полу кузова → напольные места.
2. **Вертикаль** на каждом месте — однородная стопка одного типа; число единиц по высоте:
   - Entschachtelt: `floor(H_кузова / H)`, ≤ `maxTiers`;
   - Verschachtelt: max `k` при `H + (k−1)·Δh ≤ H_кузова`, ≤ `maxNested`.
3. Итог = места × ярусы, распределённые по количествам заявки.

Допущение: одно напольное место = стопка одного типа (типы не смешиваются по высоте).

Отклонено: полный 3D-упаковщик произвольных грузов (сложнее без пользы для этой задачи);
чистый 2D без вертикали (не отражает Verschachtelt/Entschachtelt — суть инструмента).

## Последствия

- (+) Резко проще и быстрее полного 3D; напрямую отражает Verschachtelt/Entschachtelt.
- (+) Разделяет ответственность: 2D-упаковщик и вертикальный расчёт независимы и тестируемы.
- (−) Смешивание разных типов в одной вертикальной стопке недоступно (приемлемо для MVP).
- (−) Не покрывает произвольные 3D-грузы — сознательно вне объёма MVP.

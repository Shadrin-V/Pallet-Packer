# ADR 008 — Интеграция @shadrin-v/engine в Lovable

Статус: Решено — см. [ADR 010](010-lovable-private-npm.md) · Дата: 2026-07-09

## Контекст

Движок `@shadrin-v/engine` спроектирован как изоморфный TS-пакет, исполняемый в браузере
([ADR 001](001-headless-ts-engine-in-browser.md)). Нужно подтвердить, что Lovable фактически
может подключить и вызвать пакет; иначе браузерная модель и контракт `0.1.0` как чисто
клиентский — под вопросом.

## Рассматриваемые варианты

1. **npm-публикация** — `@shadrin-v/engine` публикуется (npm или приватный реестр), Lovable импортирует.
2. **Vendoring** — исходники движка включаются в проект Lovable напрямую.
3. **REST-fallback** — движок за Node HTTP-обёрткой; Lovable вызывает REST (пересматривает ADR 001).

## Решение

Открыто. Принимается по итогам спайка «Lovable import check» (`LKWkalk-qrd.16`).

## Последствия

- Результат определяет, останется ли расчёт браузерным (варианты 1–2) или потребуется REST-слой
  (вариант 3); влияет на [ADR 001](001-headless-ts-engine-in-browser.md) и границы
  [api-contract.md](../api-contract.md) `0.1.0`.
- Блокирует UI-задачу `LKWkalk-qrd.13` до выбора способа интеграции.

# /start — приход за машину

Восстанови **истину** и контекст ПЕРЕД работой. Одна «книжная подпорка» сессии (пара к `/wrap`).
Иди по шагам; заблокированный шаг = СТОП + точный отчёт, не выдумывай состояние.

## P1. Стартуй от истины (синк VCS + трекер с remote)
1. `git status` — есть незакоммиченные локальные правки? СТОП, покажи их (не затирай), спроси.
2. `git pull --ff-only` — подтянуть `main`. Не fast-forward (расхождение)? СТОП, покажи, спроси.
3. `bd dolt pull` — подтянуть beads-БД из git-remote (`refs/dolt/data`).

## P2. Восстанови контекст/память
4. `bd prime` — рабочий контекст beads + персистентные памяти (`bd remember`). Это ОБЯЗАТЕЛЬНО после
   смены машины/очистки контекста. (Вторично: харнесс сам подгружает индекс памяти — читай при нужде;
   но истина проекта — в beads, см. CLAUDE.md.)

## P3. Знай, что можно брать
5. `bd ready` — доступная работа без блокеров; `bd list --status in_progress` — что уже в работе.
6. `git log --oneline -5` — последние коммиты; `git rev-parse --short origin/main` — синхронна ли `main`.

## P4. Не доверяй устаревшим входным докам
7. Прочитай СВЕЖИЙ handover — последний по дате в имени: `ls docs/superpowers/HANDOVER-* | tail -1`.
   - ПРОПУСТИ раздел/док с баннером `SUPERSEDED` или про уже закрытую задачу (сверь `bd show <id>`).
   - Особое внимание: разделы «Грабли» и «Что взять дальше».

## P5. Отчёт «где остановились»
8. Кратко (5–8 строк): на чём остановились, состояние прода/гейтов, кандидаты из `bd ready`.
   Если неочевидно, с чего начать — задай ОДИН уточняющий вопрос, не сыпь списком.

Опц. прод-статус: `curl -s https://ladungsplaner.holz-schaefer.de/api/health` → `{"status":"ok","contract":"…"}`.

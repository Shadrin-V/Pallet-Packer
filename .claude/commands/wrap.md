# /wrap — уход с машины

Вызов `/wrap` = **ЯВНАЯ авторизация на commit/push** (перекрывает «не коммить без просьбы»).
Порядок СТРОГИЙ: **сначала реконсиляция (§1), потом фиксация (§2).** §1 — главная ценность команды,
её нельзя пропускать. Заблокированный шаг = СТОП + точный отчёт (команда + ошибка), не имитировать успех.

---

## §1. РЕКОНСИЛЯЦИЯ doc↔реальность (ГЛАВНОЕ)

**Корневая проблема:** один факт (имя поля/константы, число, номер задачи, статус, версия контракта,
счётчик тестов) живёт в N местах, а правится там, где «родился» — особенно поздний фикс (на
тесте/ревью/стенде) оседает в коде/отчёте, а копии в доках расходятся. Плюс forward-доки
(«осталось X», «уже обновлено») пишутся как свершившийся факт и не корректируются.

**Железное правило:** доказательство ПРЕЖДЕ утверждения — на каждое «согласовано/обновлено» приложи
выхлоп поиска (`git grep`), а не «я помню, что поправил».

**Ход 1 — Дельта сессии.** Собери:
- `git log --oneline <начало-сессии>..HEAD` — что смержено/закоммичено;
- `bd list` / закрытые и изменённые задачи — статусы;
- какие файлы кода тронуты; какие РЕШЕНИЯ/ФИКСЫ приняты — **особенно найденные поздно**.

**Ход 2 — Один факт → все копии.** Для КАЖДОГО изменённого факта ищи по СТАРОМУ значению и приведи к
одному во всех местах, где он может жить:
- код: `packages/*`, `apps/*`;
- доки: `docs/spec.md`, `docs/design.md`, `docs/api-contract.md`, `docs/adr/`, `docs/CHANGELOG.md`,
  `docs/superpowers/specs/`, `docs/superpowers/plans/`, `docs/superpowers/HANDOVER-*`;
- трекер/память: `bd show <id>`, `bd memories`.
- Команда: `git grep -n "<старое-значение>"` по всему дереву (имя, число, id, статус, версия).
- **Поздний фикс пропагируй ВВЕРХ по иерархии** (дизайн → спека → план → handover), не только в код/отчёт.

**Ход 3 — Убей устаревшие входные точки и ложные обещания.** Док, называющий себя «точкой входа» /
«осталось» / «pending» / «TODO» / «будет» / «уже сделано» про работу, которая на деле сделана (или ещё
нет) → баннер `> ⚠️ SUPERSEDED (<абсолютная дата>): актуально — <ссылка>` или удаление. Убери
инструкции, ведущие к УЖЕ исправленному багу. Худший дрейф — устаревший handover, называющий
исправленный баг: следующая сессия внесёт его заново.

**Ход 4 — Гигиена памяти.** `bd memories` отражают новую/закрытую работу; ни одна память не
противоречит сама себе (поздний ответ vs ранний черновик — зачеркни черновик); каждое заявленное
«обновлено» реально обновлено. Правь через `bd remember --key <key> "…"`, удаляй `bd forget <key>`.
(Проект: персистентная память — в beads, НЕ markdown-TODO/MEMORY.md, см. CLAUDE.md.)

## §2. Фиксация

**W2 — Хвост в трекер.** Всё незавершённое → `bd create` (+ `bd dep add` для связей), НЕ в голову и
НЕ в markdown-TODO.

**W3 — Гейты + доказательства** (если менялся код). КОРНЕВЫЕ команды (не workspace-scoped!):
- `npm run typecheck` · `npm run lint` (= `eslint .`) · `npm test` (= `vitest run`, весь монорепо).
- НЕ писать «green», не увидев вывод. Браузерная проверка — только настоящий Chrome (jsdom врёт).

**W4 — Обнови handover.** Новый `docs/superpowers/HANDOVER-<YYYY-MM-DD>[-topic].md` (АБСОЛЮТНАЯ дата):
что сделано · решения/ФИКСЫ · **Грабли** · Состояние (`main`/прод/гейты/beads) · «Что взять дальше» ·
локальный запуск. Handover — это ВХОД для `/start`; он должен быть ПРАВДОЙ (сверен в §1).

**W5 — Зафиксируй всё.** Порядок: `git add` + `git commit` → `bd dolt push` (синк трекера) →
`git push origin main`. Дерево после — чистое (`git status`), `origin/main == main`.
- ⚠️ **push в `main` = выкладка на прод** (CD, ADR 023; Coolify ~4 мин; обязательный статус-чек `ci`).
  «По правилу» — через PR, чтобы `ci` был гейтом ДО мержа (прямой push обходит гейт по bypass-правам).
- Заблокированный/отклонённый шаг = СТОП + точная команда и ошибка, не выдумывать успех.

Финал: `curl -s https://ladungsplaner.holz-schaefer.de/api/health` + короткий отчёт что
закоммичено/запушено, состояние прода, `bd ready` на следующую сессию.
